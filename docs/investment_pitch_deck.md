## スライド3: 市場機会

### カスタマーサポート自動化市場
- **世界市場規模**: 約$89億（2023年）→ $240億（2030年予測）
- **CAGR**: 約24%
- **日本市場**: 約1,200億円（2023年）→ 3,500億円（2030年予測）

### 市場の課題
1. **人材不足**: サポート担当者の採用・育成が困難
2. **コスト高**: 24時間対応には高い人件費が必要
3. **品質のばらつき**: 担当者によって回答品質が異なる
4. **多言語対応**: グローバル展開時の言語対応が困難

### ターゲット市場
- **一次市場**: 中小〜中堅企業（従業員50-500名）
  - サポートチームが限られている
  - 夜間・休日対応に課題
- **二次市場**: 大企業（従業員500名以上）
  - グローバル展開企業
  - 24時間対応が必要なサービス業

## スライド4: 問題提起

### 企業が抱えるサポート課題

| 課題 | 現状 | 影響 |

| **導入コスト** | 開発リソースが必要 | 導入ハードルが高い |

### 従来のチャットボットの限界
- ❌ 回答精度が日によってバラバラ
- ❌ ブランドらしさが失われる
- ❌ 結局サポート工数は減らない
- ❌ 導入・カスタマイズに時間がかかる

## スライド7: 技術的優位性

### 技術スタック

**フロントエンド**
- Next.js 13（React 18）
- TypeScript
- Tailwind CSS

**バックエンド**
- LangChain（AI/LLMフレームワーク）
- OpenAI GPT（チャット生成）
- Supabase（PostgreSQL + pgvector）
- Redis + BullMQ（ジョブキュー）

**インフラ**
- Vercel（ホスティング）
- Upstash Redis（キュー管理）
- Supabase（データベース）

### 技術的差別化ポイント

1. **スケーラブルなアーキテクチャ**
   - Redis + BullMQによる非同期処理
   - 大量の学習ジョブにも対応可能

2. **効率的なベクトル検索**
   - Supabase pgvectorによる高速検索
   - コスト効率的な埋め込み管理

3. **柔軟なモデル選択**
   - プラン別にモデルを最適化
   - コストと品質のバランス調整

4. **差分更新機能**
   - サイト更新時のみ再学習
   - コスト削減と高速化

5. **堅牢なセキュリティ**
   - RLS（Row Level Security）によるデータ隔離
   - 暗号化とアクセス制御
